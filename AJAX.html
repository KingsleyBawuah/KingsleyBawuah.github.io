AJAX

Along with an explanation of JSON

Ajax stands for Asynchronous JavaScript and and XML, and it allows you to do cool things like reload a part of page in the background without the page needing a full refresh . When AJAX was named API's used XML primarily for exporting data, but now a days JSON (Javascript Standard Object Notation) is used more commonly.

Whats that? another acronym? You're absolotely right. JSON is basicallly a standard format for displaying interchangable data. It is used because it easier for humans to read compared to XML (depending on who you ask), while still being easy for machines to parse and display. JSON, much like objects in JavaScript objects consist of "name" and "value" pairs. However unlike JavaScript , JSON objects require each name must(BOLD) be a string, and they cannot store functions the way objects can in JavaScript.

Enough background knowledge, lets begin on our application. First things first we need an account on AniList. Once created navigate to the settings, then to the developer tab. Here is where we will register our application to recieve access to AniLists API. This is done as a method of identification and authentication in which we will go into more detail in the following sections. For now fill in the Client Redirect URL with any URL (we will change this later) and a simple name for your application. We will change these in the future so don't worry to much about them now.
